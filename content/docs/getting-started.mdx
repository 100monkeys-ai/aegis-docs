# Getting Started

AEGIS manages the full lifecycle of AI agents — from manifest deployment through iterative execution, secure tool access, workflow coordination, and multi-agent swarms. Agents run in isolated containers, tools are proxied through a signed security layer, and every execution is observed, scored, and learned from.

This guide walks you through installing AEGIS, starting the local stack, and running your first agent execution.

---

## Install the AEGIS Binary

Download the latest release for your platform from the [GitHub Releases page](https://github.com/100monkeys-ai/aegis-orchestrator/releases).

```bash
curl -fsSL https://github.com/100monkeys-ai/aegis-orchestrator/releases/latest/download/aegis-linux-x86_64.tar.gz \
  | tar -xz -C /usr/local/bin
```

The release archive contains a single `aegis` binary that will be extracted to `/usr/local/bin`. Verify the installation:

```bash
aegis --version
```

---

## Clone aegis-examples

The `aegis-examples` repository contains the Docker Compose stack for all backing services (PostgreSQL, SeaweedFS, Temporal, Keycloak, Ollama, Cortex) and pre-built demo agent manifests.

```bash
git clone https://github.com/100monkeys-ai/aegis-examples.git && cd aegis-examples
```

---

## Start the Local Stack

Copy the environment file and start all backing services:

```bash
cp deploy/.env.example deploy/.env
docker compose -f deploy/docker-compose.yml up -d
```

This starts PostgreSQL, SeaweedFS, Temporal, Keycloak, Ollama, and the Cortex services. Wait ~30 seconds for Temporal to finish initializing. You can check the status with:

```bash
docker compose -f deploy/docker-compose.yml ps
```

All services should show `running` (healthy).

---

## Pull the Default Model

Pull the default Ollama model that the example agents use:

```bash
docker compose -f deploy/docker-compose.yml exec ollama ollama pull phi3:mini
```

This downloads the `phi3:mini` model into the Ollama container. The example agents are configured to use this model by default.

---

## Deploy and Run the hello-world Agent

Deploy the hello-world agent:

```bash
aegis agent deploy ./agents/hello-world/agent.yaml
```

Execute the agent with a task:

```bash
aegis task execute hello-world \
  --input '{"task": "Write a Python function that returns the Fibonacci sequence up to n."}' \
  --follow
```

The `--follow` flag streams iteration events to the terminal as they happen. You'll see output similar to:

```
2026-02-25T13:56:43.091943Z  INFO Delegating to daemon API
Executing agent 88b73d1b-0da0-4b92-9376-fd744b9cafbf...
✓ Execution started: f668f593-370f-4c19-b043-0487e9bd1ae5
[2026-02-25T13:56:43.140898+00:00] Execution started
[2026-02-25T13:56:43.164238272+00:00] Iteration 1
[2026-02-25T13:57:34.455428513+00:00] LLM [default]
[STDOUT] "First, I would write the required `fib_sequence` function in `/workspace/solution.py`:
```

The agent will generate code, run validation, and either succeed or refine across multiple iterations until it produces a working solution.

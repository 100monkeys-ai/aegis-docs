# CLI Reference

The `aegis` binary is the primary interface for managing the AEGIS daemon and all resources.

## Global Flags

These flags are available on all commands:

| Flag | Environment Variable | Default | Description |
|---|---|---|---|
| `--config <path>` | `AEGIS_CONFIG_PATH` | (auto-discovered) | Path to the daemon configuration file. |
| `--host <host>` | `AEGIS_HOST` | `127.0.0.1` | Hostname of the AEGIS daemon HTTP API. |
| `--port <port>` | `AEGIS_PORT` | `8000` | Port of the AEGIS daemon HTTP API. |
| `--log-level <level>` | `AEGIS_LOG_LEVEL` | `info` | Log verbosity: `trace`, `debug`, `info`, `warn`, `error`. |
| `--daemon` | — | `false` | Start the process as a background daemon service rather than running a CLI command. |

The daemon HTTP API runs on port `8000` by default (see [ADR-026](/docs/architecture/adr-026)).

## `aegis daemon`

Manage the AEGIS daemon process.

### `aegis daemon start`

Start the daemon in the foreground.

```bash
aegis daemon start
aegis daemon start --config /etc/aegis/config.yaml
```

### `aegis daemon stop`

Stop the running daemon.

```bash
aegis daemon stop
```

### `aegis daemon status`

Show daemon status, version, uptime, and connected dependency health.

```bash
aegis daemon status
```

### `aegis daemon install`

Install the daemon as a system service (systemd).

```bash
aegis daemon install
```

### `aegis daemon uninstall`

Remove the system service installation.

```bash
aegis daemon uninstall
```

## `aegis agent`

Manage agent definitions.

### `aegis agent deploy <manifest>`

Deploy or update an agent from a manifest file. Re-deploying an agent with the same `metadata.name` updates the manifest in-place.

```bash
aegis agent deploy ./agent.yaml
aegis agent deploy ./agent.yaml --validate-only
```

| Flag | Default | Description |
|---|---|---|
| `--validate-only` | `false` | Parse and validate the manifest without persisting it. |

### `aegis agent list`

List all deployed agents.

```bash
aegis agent list
```

### `aegis agent show <AGENT_ID>`

Show the full stored manifest for an agent. `<AGENT_ID>` must be a UUID.

```bash
aegis agent show a1b2c3d4-0000-0000-0000-000000000001
```

### `aegis agent remove <AGENT_ID>`

Remove a deployed agent. Running executions are not affected.

```bash
aegis agent remove a1b2c3d4-0000-0000-0000-000000000001
```

### `aegis agent logs <AGENT_ID>`

Stream event log output for a specific agent.

```bash
aegis agent logs a1b2c3d4-0000-0000-0000-000000000001
aegis agent logs a1b2c3d4-0000-0000-0000-000000000001 --follow
aegis agent logs a1b2c3d4-0000-0000-0000-000000000001 --errors
aegis agent logs a1b2c3d4-0000-0000-0000-000000000001 --verbose
```

| Flag | Default | Description |
|---|---|---|
| `--follow` | `false` | Tail log output. |
| `--errors` | `false` | Show error-level events only. |
| `--verbose` | `false` | Include LLM prompt and response content. |

## `aegis task`

Manage agent task executions.

### `aegis task execute <AGENT>`

Start a single agent execution. `<AGENT>` may be an agent UUID or a path to a manifest file.

```bash
aegis task execute a1b2c3d4-0000-0000-0000-000000000001
aegis task execute ./agent.yaml
aegis task execute a1b2c3d4-0000-0000-0000-000000000001 --input '{"task": "Write a primality check."}'
aegis task execute a1b2c3d4-0000-0000-0000-000000000001 --input '{"task": "..."}' --follow
```

| Flag | Default | Description |
|---|---|---|
| `--input <json>` | `null` | JSON input string passed to `bootstrap.py` via `task.input`. Accepts an inline JSON string or `@file.json` to read from a file. |
| `--wait` | `false` | Block until the execution completes. |
| `--follow` | `false` | Stream iteration events to stdout (implies `--wait`). |

### `aegis task status <EXECUTION_ID>`

Show the current status of an execution.

```bash
aegis task status a1b2c3d4-1111-0000-0000-000000000001
```

### `aegis task logs <EXECUTION_ID>`

Stream or print iteration logs.

```bash
aegis task logs a1b2c3d4-1111-0000-0000-000000000001
aegis task logs a1b2c3d4-1111-0000-0000-000000000001 --follow
aegis task logs a1b2c3d4-1111-0000-0000-000000000001 --errors-only
aegis task logs a1b2c3d4-1111-0000-0000-000000000001 --verbose
```

| Flag | Default | Description |
|---|---|---|
| `--follow` | `false` | Tail log output until the execution ends. |
| `--errors-only` | `false` | Suppress non-error output lines. |
| `--verbose` | `false` | Include LLM prompt and response text in output. |

### `aegis task cancel <EXECUTION_ID>`

Cancel a running execution.

```bash
aegis task cancel a1b2c3d4-1111-0000-0000-000000000001
aegis task cancel a1b2c3d4-1111-0000-0000-000000000001 --force
```

| Flag | Default | Description |
|---|---|---|
| `--force` | `false` | Kill without waiting for graceful shutdown. |

### `aegis task remove <EXECUTION_ID>`

Delete an execution record from the database.

```bash
aegis task remove a1b2c3d4-1111-0000-0000-000000000001
```

### `aegis task list`

List recent executions.

```bash
aegis task list
aegis task list --agent-id a1b2c3d4-0000-0000-0000-000000000001
aegis task list --limit 50
```

| Flag | Default | Description |
|---|---|---|
| `--agent-id <uuid>` | (all agents) | Filter executions by agent UUID. |
| `--limit <n>` | `20` | Maximum number of results to return. |

## `aegis workflow`

Manage workflow definitions.

### `aegis workflow validate <file>`

Parse and validate a workflow manifest without deploying it.

```bash
aegis workflow validate ./pipeline.yaml
```

### `aegis workflow deploy <file>`

Deploy a workflow to the registry.

```bash
aegis workflow deploy ./pipeline.yaml
```

### `aegis workflow run <name>`

Start a workflow execution.

```bash
aegis workflow run dev-pipeline
aegis workflow run dev-pipeline --input '{"task": "..."}'
aegis workflow run dev-pipeline --input '{"task": "..."}' --follow
aegis workflow run dev-pipeline --param branch=main --param env=staging
```

| Flag | Default | Description |
|---|---|---|
| `--input <json>` | `null` | JSON input available as Blackboard context. |
| `--param <key=value>` | — | Individual named parameters. May be repeated. |
| `--follow` | `false` | Stream state-transition events to stdout. |

### `aegis workflow list`

List all registered workflows.

```bash
aegis workflow list
aegis workflow list --long
aegis workflow list --label team=platform
```

| Flag | Default | Description |
|---|---|---|
| `--long` | `false` | Show detailed information (states, labels). |
| `--label <key=value>` | — | Filter by label. May be repeated. |

### `aegis workflow describe <name>`

Show the full YAML definition of a deployed workflow.

```bash
aegis workflow describe dev-pipeline
aegis workflow describe dev-pipeline --output json
```

| Flag | Default | Description |
|---|---|---|
| `--output <format>` | `yaml` | Output format: `yaml` or `json`. |

### `aegis workflow logs <EXECUTION_ID>`

Stream or print workflow execution logs.

```bash
aegis workflow logs a1b2c3d4-2222-0000-0000-000000000001
aegis workflow logs a1b2c3d4-2222-0000-0000-000000000001 --follow
aegis workflow logs a1b2c3d4-2222-0000-0000-000000000001 --transitions
```

| Flag | Default | Description |
|---|---|---|
| `--follow` | `false` | Tail log output until execution ends. |
| `--transitions` | `false` | Show only state-transition events. |

### `aegis workflow delete <name>`

Delete a workflow from the registry.

```bash
aegis workflow delete dev-pipeline
aegis workflow delete dev-pipeline --yes
```

| Flag | Default | Description |
|---|---|---|
| `--yes` | `false` | Skip the confirmation prompt. |

## `aegis config`

Utility commands for the daemon configuration file.

### `aegis config show`

Print the resolved daemon configuration, with `env:` variables expanded and secrets redacted.

```bash
aegis config show
aegis config show --paths
```

| Flag | Default | Description |
|---|---|---|
| `--paths` | `false` | Also show the config file search paths that were checked. |

### `aegis config validate`

Validate an `aegis-config.yaml` file without starting the daemon.

```bash
aegis config validate
aegis config validate ./my-config.yaml
```

### `aegis config generate`

Generate a commented sample `aegis-config.yaml`.

```bash
aegis config generate
aegis config generate --output /etc/aegis/config.yaml
aegis config generate --examples
```

| Flag | Default | Description |
|---|---|---|
| `--output <path>` | `./aegis-config.yaml` | Output file path. |
| `--examples` | `false` | Include inline comments and example values. |

## `aegis update`

Apply pending database schema migrations. Run this after upgrading the `aegis` binary.

```bash
aegis update
aegis update --config /etc/aegis/config.yaml
```

## Exit Codes

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | Resource not found |
| `2` | Validation error (manifest, config) |
| `3` | Permission denied (IAM role insufficient) |
| `4` | Resource conflict (e.g., agent name already exists) |
| `5` | Connection error (daemon unreachable) |
| `10` | Unhandled internal error |
